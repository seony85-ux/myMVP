# Updated Wireframe Guide

## 1. 인트로 화면 (Intro)

**목적**: 서비스 분위기 전달 및 루틴 시작 진입점 제공

* **상단**: 태그라인 "잠시, 피부에만 집중하는 시간"
* **중앙**: 원형 이미지 + 안내 문구 "짧은 루틴을 시작하세요"
* **하단**: CTA 버튼 [오늘의 루틴 고르기]

---

## 2. 루틴 설정 화면 (Emotion + BGM + Routine Mode + Skincare)

**목적**: 루틴 시작 전 설정을 빠르고 직관적으로 완료

### 상단: 제목 "오늘의 루틴 설정"

### 섹션 1: 감정 선택

* 질문: "지금 기분은 어떤가요?"
* UI: 이모지 기반 1~5 점수, 선택 시 하이라이트, 라벨 표시

### 섹션 2: BGM 선택

* 제목: "배경음"
* 카드형 BGM 리스트 (3개 + 없음)
* 이미지 + 제목 + 분위기 설명 포함

### 섹션 3: 루틴 모드 선택

* 제목: "루틴 모드"
* 선택지: 기본 / 단계별 가이드 (Toggle 또는 라디오 버튼)

### 섹션 4: 스킨케어 단계 선택 (루틴 모드: 단계별 가이드인 경우만 표시)

* 제목: "스킨케어 단계"
* 선택 UI: 토글 버튼 (토너 / 에센스 / 크림)
* 최소 1개 선택 필수, 미선택 시 경고 문구 표시

### 하단: CTA 버튼 [다음]

* 유효성 검사: 필수 항목 미선택 시 비활성화 또는 에러 메시지

---

## 3. 음성 가이드 설정 화면

**목적**: 음성 가이드 사용 여부 설정

* **상단**: 질문 "음성 가이드를 들으시겠어요?"
* **중앙**: On/Off 토글 (기본값: On)

  * 토글 아래 상태별 설명 표시
* **하단**: CTA 버튼 [루틴 시작]

---

## 4. 루틴 진행 화면

**목적**: 스킨케어+명상 루틴 단계별 자동 진행

* **상단**: 현재 단계명 (예: "에센스 단계")
* **진행 상태 시각화**: 점(dot) 인디케이터 (완료, 현재 단계 구분)
* **중앙**: 현재 명상 문구 (전환 시 페이드 효과 가능)
* **하단**: 
  * 마지막 단계: [끝내기] 버튼
  * 진행 중: [일시중지]/[재개] 버튼 및 [중단하기] 버튼 (나란히 배치)

### 개발용 옵션 (dev=1 파라미터 시만 노출)

* DEV 버튼: [다음 단계로], [루틴 완료], [중단]
* 화면 우측 하단에 고정 노출
* 우측 상단에 현재 설정 상태 표시 (음성 가이드, BGM)

---

## 5. 감정 변화 기록 화면 (After Emotion)

**목적**: 루틴 후 감정 상태 입력 및 비교

* **상단**: 질문 "지금 기분은 어떤가요?"
* **서브텍스트**: "이전: N점" 표시
* **중앙**: 이모지 기반 감정 점수 선택 (1~5), 라벨 포함
* **하단**: CTA 버튼 [결과 보기]
* **에러 처리**: 미선택 시 경고 표시 및 버튼 비활성화

---

## 6. 결과 요약 화면 (Summary)

**목적**: 감정 변화 시각화 및 사용자 피드백 수집

* **상단**: 제목 "오늘의 루틴 요약"

### 섹션 1: 감정 변화

* 숫자 비교 (예: 3 → 5), 그래픽 시각화 컴포넌트 사용

### 섹션 2: 만족도 선택

* 질문: "이번 루틴에 만족하셨나요?"
* 이모지 또는 점수(1~5) UI
* 에러 시 경고 텍스트

### 섹션 3: 재사용 의향 선택

* 질문: "다시 사용하시겠어요?"
* Yes/No 토글 또는 버튼
* 미선택 시 경고

### 하단: CTA 버튼 [완료]

* 모든 입력 완료 시 활성화

### 중단(aborted) 세션 처리

* URL 파라미터 `?aborted=1`에 따라 문구 또는 처리 변경 (TODO)

## 7. 감사 화면 (Thank-you)

**목적**: 루틴 완료 후 긍정적 마무리

* **중앙**:

  * 원형 이미지
  * 메인 텍스트: "체험해주셔서 감사합니다"
  * 서브 문구: "매일 조금씩, 나에게 집중하는 순간을..."

* **하단 버튼 1**: [처음으로] → 인트로 이동

* **하단 버튼 2**: [더 알아보기] → 현재는 placeholder, 추후 연결 예정
